# Neuron Node Builder - Project Overview

## Purpose
Neuron Node Builder is a Node-RED fork for machine-to-machine commerce powered by Hedera blockchain technology. It provides custom nodes for buyers and sellers to interact with the Neuron network, enabling blockchain-based IoT and device communication.

## Tech Stack

### Core Technologies
- **Node.js** v18.5+ (required for Node-RED 4.x)
- **Node-RED** 4.x (included in package)
- **Go** v1.19+ (for building SDK dependencies)
- **Hedera Blockchain** (testnet/mainnet support)
- **Ethereum/EVM** integration

### Key Dependencies
- **@hashgraph/sdk** - Hedera SDK
- **ethers.js** - Ethereum integration
- **Express** - Web server
- **MQTT** - IoT messaging protocol
- **neuron-js-registration-sdk** - Custom Neuron SDK
- **Playwright** - Visual testing framework
- **Grunt** - Build system

### External Go Dependencies (built from source)
1. **neuron-go-hedera-sdk** - Go Hedera SDK
2. **neuron-nodered-sdk-wrapper** - Node-RED SDK wrapper

## Supported Platforms
- **macOS** (Intel x64 and Apple Silicon ARM64)
- **Windows** (x64)
- **Linux support removed** (as of recent updates)

## Project Structure

### Main Directories
- `neuron/` - Custom Neuron-specific code
  - `nodes/` - Custom Node-RED nodes (buyer, seller, GPT, P2P, etc.)
  - `services/` - Core services (SDK resolver, environment, updates, health monitoring)
  - `routes/` - Express routes
  - `pages/` - Web pages
  - `lib/` - Libraries
  - `theme/` - UI theme customization
  - `userdir/` - User data directory
  
- `packages/node_modules/` - Node-RED core packages
  - `node-red/` - Main Node-RED runtime
  - `@node-red/editor-client/` - Web editor
  - `@node-red/nodes/` - Standard nodes

- `tests/` - Test suites
  - `visual/` - Playwright visual regression tests
  - `unit/` - Unit tests
  - `nodes/` - Node tests

- `.github/workflows/` - CI/CD pipelines
  - `build-and-sign-production.yml` - Production build & signing
  - `visual-ai-regression.yml` - Visual testing
  - `tests.yml` - Unit tests
  - `release.yml` - Release automation

- `build/` - Build outputs and packaged executables
- `scripts/` - Build and utility scripts
- `public/` - Compiled static assets (generated by grunt build)

## Custom Nodes
- **buyer** - Create buyer devices and consume data
- **seller** - Create seller devices and publish data
- **neuron-gpt** - GPT integration
- **neuron-p2p** - Peer-to-peer communication
- **connection-status** - Monitor connection status
- **stdin/stdout/stderr** - Process I/O handling

## Key Configuration Files
- `neuron-settings.js` - Custom Node-RED settings
- `.env` - Environment variables (Hedera credentials, SDK paths)
- `package.json` - Project dependencies and scripts
- `Gruntfile.js` - Build configuration
- `playwright.config.ts` - Visual test configuration
- `.jshintrc` - JavaScript linting rules
- `entitlements.plist` - macOS code signing entitlements
